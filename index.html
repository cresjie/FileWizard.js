Light javascript library for drag and drop upload files. Provides skeletal support to your file upload. You can easily create your theme by just attaching in the triggered events.

Requirements

  1. jQuery
  2. FileUploader (already included in the FileWizard-bundle.min.js)

Bower

bower install filewizard.js --save

Installation

bower install filewizard.js --save

Advance (Multiple/Auto-upload)

Constructor

FileWizard(element, options, headers)

  • element String[selector], jQuery element, DOM

  • options Object

  • headers Object

Options

  • url String| required

    - url to where to send the request

  • method String | default: POST

    - http method to use in the request

  • progress Function(percent)

    - event listener for progress event

  • abort Function

    - triggers when the request is aborted

  • success Function(response, event)

    - triggers when the request is success: status 200

  • error Function(response, event)

    - triggers when error occur in the request

  • complete Function(response, event)

    - triggers either the request is error or success

  • dragover Function

    - mouse event drag over the element

  • drop Function

    - on drop item event

  • dragenter Function

    - ragenter mouse event

  • rejected Function(file,error)

    - callback if the file is rejected (e.g file_limit, file_type)

  • fileAdded Function(file)

    - event callback if the file is successfully added to the list

  • fileRemoved Function(Array files)

    - trigger when file(s) is/are removed

  • paramName String| default: files

    - paramater name for the file

  • clickable Boolean| default: true

    - opens window file browser if the element is clicked

  • acceptedFiles Array | default: ['jpg','jpeg','png','gif']

    - accepted file extension

  • maxSize Integer | default: 5

    - maximum size of the file in MB

  • multipleFiles Boolean | default: true

    - upload multiple file in one request

  • beforeSubmit Function(settings, Array|File file(s) )

    - triggers before the request is submitted

  • submitted Function(settings, Array|File file(s) )

    - triggers after the request is submitted

  • data Object

    - additional HTTP parameters

Headers / Defaults

  • Cache-Control: no-cache

  • X-Requested-With: XMLHttpRequest

  • see HTTP Header list for more info.

Methods

  • addData Function(key, value) or Function(Object)

    - add data paramater to the request in a key-value pair. e.g fw.addData({x: 0, y: 0, width: 300, height: 300}); //data parameter for the image to be cropped

  • getFiles()

    - returns files

  • resetFiles()

    - resets file list

  • addFiles Function([FileList|Array] files)

    - add files to the list

  • removeFile Function(index, range) or Function(file)

    - remove file(s) in the list

  • setOptions Function(Object option)

    - set the options of the file wizard

  • send Function(Object data) *optional

    - send the data/files to the url specified, optional additional data parameters can be added before send

  • abort Integer | default: 0

    - cancel's the request at the specific index

  • abortAll

    - cancel all the ongoing request
